# ব্যবহারকারী নির্দেশিকা

এই নির্দেশিকা সাধারণ ব্যবহারকারীর জন্য। এখানে technical implementation নয়, app ব্যবহারের মূল নিয়ম রাখা হয়েছে।

## প্রবেশ

- মূল প্রবেশপথ: `index.html`.
- দায়িত্ব অনুযায়ী login card নির্বাচন করুন।
- যাদের login ID লাগে, তারা login ID এবং PIN দিয়ে প্রবেশ করবেন।
- যাদের শুধু PIN লাগে, তারা নির্ধারিত PIN দিয়ে প্রবেশ করবেন।
- কাজ শেষে logout ব্যবহার করুন।

## প্রধান ভূমিকা

| ভূমিকা | কাজের ধরন |
| --- | --- |
| জিম্মাদার / admin | সব module দেখা, settings ও অনুমতি পরিচালনা |
| সহকারী জিম্মাদার | admin প্রদত্ত সীমিত module/settings access |
| দফতর দায়িত্বশীল | হাজিরা, ছাত্র, হিসাব, কর্মসূচি, দফতর সংশ্লিষ্ট কাজ |
| বর্ষ দায়িত্বশীল / শিক্ষক | নিজ বর্ষের ছাত্র, হাজিরা, দর্স/রিপোর্ট সংশ্লিষ্ট কাজ |
| বিভাগ দায়িত্বশীল | বিভাগের আয়-ব্যয়, মজুদ, রিপোর্ট, edit request |
| হিফজ দায়িত্বশীল | হিফজ group, সদস্য, অগ্রগতি ও activity |
| খেদমত দায়িত্বশীল | খেদমতে খালক সংশ্লিষ্ট তথ্য ও হিসাব |

## মাদরাসা Module

মাদরাসা অংশে সাধারণত নিচের কাজগুলো থাকে:

- ছাত্র তালিকা দেখা ও ছাত্রের বিস্তারিত modal খোলা।
- ছাত্রের ছবিতে click করলে বড় preview দেখা।
- হাজিরা নেওয়া ও অনুপস্থিত তালিকা দেখা।
- দফতর হিসাব, বকেয়া, রিপোর্ট এবং settings দেখা।
- কর্মসূচি পরিচালনা, যেমন কোরবানি বা অন্য বিশেষ কর্মসূচি।
- দর্সের পরিমাণ ও ইতিহাস দেখা।
- পরীক্ষা, হিফজ, লাইব্রেরি, পুরনো ছাত্র এবং বর্ষ উন্নীতকরণ সংশ্লিষ্ট কাজ।

## কর্মসূচি

- দফতর দায়িত্বশীলের হিসাব অংশের ভেতরে কর্মসূচি পাওয়া যাবে।
- প্রতিটি কর্মসূচির আয়ের ধরন, ব্যয়ের ধরন এবং বিশেষ option আলাদা হতে পারে।
- কোনো কর্মসূচিতে হিস্যা enabled থাকলে আয় যুক্ত করার সময় হিস্যা হিসেবে চিহ্নিত করে কত হিস্যা তা লিখতে পারবেন।
- ব্যয়ের সাথে memo/receipt থাকলে attachment যুক্ত করার ব্যবস্থা থাকতে পারে।

## বিভাগ Module

বিভাগ দায়িত্বশীলরা নিজ বিভাগের কাজ করবেন:

- আয় ও ব্যয় যুক্ত করা।
- পণ্য/মজুদ দেখা ও update করা।
- report দেখা।
- পুরনো বা locked entry পরিবর্তনের জন্য edit/delete request পাঠানো।
- admin অনুমোদন করলে request কার্যকর হবে।

## Admin / Settings

Admin বা অনুমতিপ্রাপ্ত সহকারী:

- user, দায়িত্বশীল এবং permission পরিচালনা করতে পারেন।
- বর্ষ শিক্ষক নিয়োগ ও কিতাব যোগ-বিয়োগ করতে পারেন, যদি permission থাকে।
- restricted সহকারীর access তার নির্ধারিত scope অনুযায়ী সীমিত থাকবে।
- settings পরিবর্তনের আগে সংশ্লিষ্ট module ও scope ঠিক আছে কিনা দেখে নিন।

## সতর্কতা

- একই কাজ একাধিক browser tab থেকে একসাথে করলে save conflict হতে পারে।
- কোনো তথ্য দেখা না গেলে আগে refresh/login state যাচাই করুন।
- PIN বা permission পরিবর্তনের পর user-কে logout করে আবার login করতে হতে পারে।
- বড় পরিবর্তন, import বা database migration করার আগে backup/verification নিশ্চিত করুন।
