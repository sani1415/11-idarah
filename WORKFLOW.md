## ১. প্রযুক্তি স্তর (বর্তমান)

| স্তর | বিবরণ |
|------|-------|
| **ফ্রন্টএন্ড** | বিশুদ্ধ HTML + CSS + Vanilla JS |
| **ডেটা স্টোরেজ** | `localStorage` (ব্রাউজারে সংরক্ষিত, ডিভাইস-নির্দিষ্ট) |
| **ব্যাকএন্ড** | নেই — সার্ভার বা ডেটাবেজ নেই এখনো |
| **হোস্টিং** | স্থানীয় ফাইল (ভবিষ্যতে Vercel + Supabase পরিকল্পিত) |
| **ফন্ট** | Noto Sans Bengali + Noto Serif Bengali (Google Fonts) |
| **শেয়ার্ড CSS** | `style.css` + প্রতিটি পেজের নিজস্ব `<style>` ব্লক |

## ৩. লগইন ও সেশন পদ্ধতি

### কীভাবে কাজ করে

```
ব্যবহারকারী → ভূমিকা নির্বাচন → ৪ সংখ্যার পিন → সেশন সেট → সংশ্লিষ্ট পেজে যান
```

- পিন যাচাই হয় `localStorage`-এর `mm_settings` থেকে
- সফল লগইনে `sessionStorage`-এ রোল ও নাম সংরক্ষিত হয়
- ব্রাউজার বন্ধ করলে সেশন শেষ (sessionStorage মুছে যায়)

### রোল ও পিন টেবিল

| রোল | পিন উৎস | লগইন পেজ | পরবর্তী পেজ |
|-----|---------|---------|------------|
| `admin` (মুহতামিম) | `mm_settings.admin_pin` | `index.html` বা `admin-hub.html` | `admin-hub.html` |
| `teacher` (বর্ষ দায়িত্বশীল) | শিক্ষকের নিজের পিন | `index.html` | `madrasa-staff.html` |
| `daftar` | দফতরের পিন | `index.html` | `madrasa-daftar.html` |
| `hifz` | হিফজের পিন | `index.html` | `madrasa-hifz.html` |
| `library` | লাইব্রেরির পিন | `index.html` | `madrasa-library.html` |
| `alumni` | পুরনো ছাত্রের পিন | `index.html` | `madrasa-alumni.html` |
| `khedmat` | খেদমতের পিন | `index.html` | `khedmat-staff.html` |
| বিভাগ দায়িত্বশীল | বিভাগের পিন | `dept-index.html` | `dept-staff.html` |

## ৫. ডেটা প্রবাহ

### বর্তমান (localStorage ভিত্তিক)

```
পেজ লোড
  ↓
api.js / dept-api.js / khedmat-api.js / chat-api.js লোড
  ↓
localStorage থেকে ডেটা পড়া (JSON.parse)
  ↓
DOM এ রেন্ডার
  ↓
ব্যবহারকারী ফর্ম পূরণ করেন
  ↓
API ফাংশন কল (যেমন: API.Students.add(...))
  ↓
localStorage এ সংরক্ষণ (JSON.stringify)
  ↓
পেজ আবার রেন্ডার
```

### মূল API ফাইলসমূহ

| ফাইল | দায়িত্ব |
|------|---------|
| `api.js` | মাদ্রাসা — ছাত্র, বর্ষ, উপস্থিতি, লগ, পরীক্ষা, খুলুক, ফি |
| `dept-api.js` | বিভাগ — বিভাগ, লেনদেন, মজুদ, সংশোধন অনুরোধ |
| `khedmat-api.js` | খেদমত — উপকারভোগী, কার্যক্রম, তহবিল, লগ |
| `chat-api.js` | বার্তা — থ্রেড, পঠন চিহ্ন, আনরিড কাউন্ট |
| `monitor-mode.js` | মুহতামিম পর্যবেক্ষণ মোড — মিউটেশন ব্লক |

## ৬. অনুমতি মডেল

### দুটি স্তর

**১. পেজ গার্ড** — পেজে ঢোকার অনুমতি:
```javascript
// শুধু admin
if (sessionStorage.getItem('mm_role') !== 'admin') location.href = 'index.html';

// যেকোনো লগইন
if (!sessionStorage.getItem('mm_role')) location.href = 'index.html';
```

**২. অপারেশন গার্ড** — পেজে ঢুকলেও পরিবর্তন করতে পারবেন না যদি মুহতামিম হন:
```javascript
if (mmMutationBlocked(showToast)) return;
```

### মুহতামিমের অনুমতি

| কাজ | অনুমতি |
|-----|--------|
| সব মডিউল দেখা | ✅ হ্যাঁ |
| দৈনন্দিন অপারেশনাল এন্ট্রি (উপস্থিতি, লেনদেন ইত্যাদি) | ❌ না (পর্যবেক্ষণ মোড) |
| মাদ্রাসা সেটিং, পিন, শিক্ষক তালিকা | ✅ হ্যাঁ |
| বার্তা — ইনবক্স ও উত্তর | ✅ হ্যাঁ |
| বিভাগ — সংশোধন অনুমোদন/প্রত্যাখ্যান | ✅ হ্যাঁ |

## ৭. বার্তা ব্যবস্থা

```
স্টাফ (যেকোনো রোল)
    ↓ বার্তা লেখেন
chat.html → chat-api.js → localStorage (mm_chat_messages)
    ↓
শুধু মুহতামিম দেখতে পান (ইনবক্স)
    ↓ উত্তর দেন
স্টাফ দেখতে পান

⚠️  স্টাফ থেকে স্টাফে সরাসরি যোগাযোগ নেই
⚠️  প্রতিটি স্টাফের থ্রেড শুধু সেই স্টাফ ও মুহতামিমের মধ্যে
```

## ৯. ভবিষ্যৎ পরিকল্পনা (CLAUDE.md অনুযায়ী)

```
বর্তমান (প্রোটোটাইপ)          →     ভবিষ্যৎ (প্রোডাকশন)
────────────────────────────────────────────────────────
localStorage                   →     Supabase (PostgreSQL)
পেজে পেজে API ফাইল             →     RPC-ভিত্তিক API (SECURITY DEFINER)
ব্রাউজার সেশন                  →     shared_users টেবিল + PIN verify RPC
কোনো সার্ভার নেই               →     Vercel deploy
একই ডিভাইসে সব ডেটা            →     যেকোনো ডিভাইস থেকে অ্যাক্সেস
```

### পরিকল্পিত ডেটাবেজ টেবিল প্রিফিক্স

| প্রিফিক্স | মডিউল |
|----------|-------|
| `shared_*` | ইউজার, নোটিফিকেশন |
| `dept_*` | বিভাগ মডিউল |
| `mdr_*` | মাদ্রাসা মডিউল |
| `kh_*` | খেদমতে খালক মডিউল |
